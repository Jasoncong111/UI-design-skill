# Optimized Questionnaire / 优化版问卷

Use this as the default intake system. / 默认使用这套问卷。

Goal: keep the interview short, reduce overlapping questions, and still capture enough detail to write one strong final prompt. / 目标是在减少重复问题的同时，依然收集足够信息来生成一份高质量最终 prompt。

## Round 1｜Universal questions / 通用问题

Ask these first. / 先问这些问题。

### Q1. Page type / 页面类型

1. Personal website / 个人网站
2. Product website or conversion page / 产品官网或转化页
3. Application UI or dashboard / 应用界面或后台
4. Redesign an existing page / 改版现有页面

### Q2. Main goal / 主要目标

1. Present a personal brand or body of work / 展示个人品牌或作品
2. Explain a product and build trust / 解释产品并建立信任
3. Drive signups, leads, or demo bookings / 获取注册、留资或预约
4. Help users view information or complete tasks / 帮助用户查看信息或完成操作
5. Improve the quality, clarity, or conversion of an existing page / 提升现有页面的质量、清晰度或转化

### Q3. Target audience / 目标受众

1. Recruiters or hiring managers / 招聘方或面试官
2. Consumers / 普通用户
3. Clients or partners / 客户或合作方
4. Internal teams / 公司内部团队
5. Investors / 投资人
6. Developers or designers / 开发者或设计师

## Round 2｜Shared design questions / 共享设计问题

Ask these for every branch. These replace many of the older repeated branch questions. / 这些问题所有分支都要问，用来替代旧版里大量重复的问题。

### Q4. Overall page character / 页面整体气质

1. Restrained and minimal / 克制极简
2. Content-led / 内容导向
3. Product-led / 产品导向
4. Showcase-led / 展示导向

### Q5. Hero structure / 首屏结构

1. Big headline plus supporting visual / 大标题加辅助视觉
2. Split layout with text and visual side by side / 左右分栏，文字和视觉并列
3. Text-led hero / 纯文字主导首屏
4. Interface-preview-led hero / 界面预览主导首屏

### Q6. Content organization / 内容组织方式

1. A few key modules / 少量重点模块
2. Vertical section-by-section explanation / 纵向分段讲解
3. Card-based modules / 卡片化模块
4. Dense information structure / 信息密集型结构

### Q7. Information density / 信息密度

1. Very spacious with little information / 留白很多，信息很少
2. Light but sufficient / 轻量但够用
3. Complete but clean / 完整但干净
4. Dense and efficiency-first / 高密度，效率优先

## Round 3｜Shared visual system questions / 共享视觉系统问题

Ask these next. / 接着问这些问题。

### Q8. Overall color style / 页面整体颜色

1. Mostly black, white, and gray / 黑白灰为主
2. Light and clean with a cool accent / 浅色干净，带一点冷色点缀

4. Dark overall with bright highlights / 深色为主，少量亮色提亮
5. Multi-color with stronger contrast / 多种颜色，对比更强
6. I want a custom palette / 我想自定义

2. Yes, use an existing brand color / 需要，使用现有品牌色
3. Not sure yet / 还不确定

### Q10. Accent color / 强调色

2. Cyan / 青
3. Green / 绿
4. Yellow / 黄
5. Gold / 金
6. Orange / 橙

8. Purple / 紫
9. Pink / 粉

11. I want a custom accent color / 我想自定义强调色

### Q11. Typography style / 字体风格

1. Sans serif throughout / 全部使用无衬线
2. Serif headlines plus sans serif body / 衬线标题加无衬线正文
3. More editorial and designed / 更有编辑感和设计感

6. I want a custom typography direction / 我想自定义字体方向

### Q12. Imagery direction / 图片方向

1. Real photography / 真人实拍
2. Product screenshots or UI visuals / 产品截图或界面视觉
3. Abstract background treatments / 抽象背景

### Q14. Primary button style / 主按钮样式

1. Standard rounded solid button / 标准圆角实心按钮

## Why this is shorter / 为什么这版更短

The old version repeated several ideas in different branches, especially layout, hierarchy, density, button style, color, typography, motion, and device priority. / 旧版在不同分支里重复询问了很多相近的问题，尤其是布局、层级、密度、按钮、配色、字体、动效和设备优先级。

This version: / 这版做了这些优化：

- asks shared questions once / 共享问题只问一次
- asks only three branch-specific questions / 每个分支只补三个关键问题
- keeps answer options concrete / 选项更具体
- reduces ambiguity between layout, hierarchy, and emphasis / 减少布局、层级和重点之间的歧义
